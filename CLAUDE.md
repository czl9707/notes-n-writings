## Vault Structure

- `projects/` — per-project context, status, and decisions
- `blog/` — published blog posts (by developer/christian/human-being)
- `note/` — reference notes (by developer/christian/human-being)
- `fleeting/` — quick captures, unorganized thoughts
- `templates/` — templates for notes, blogs, projects
- `media/` — images and other media files

---

## Working on Projects

### Before Starting Work

1. Ask which project we're working on
2. Read `projects/<name>/STATUS.md`
3. Summarize current state, wait for instructions

### During Work

- After significant decisions, ask if I should log to DECISION.md
- When unsure, ask — don't guess

### Before Ending a Session

- Update the project's STATUS.md if progress was made
- Note any unresolved items

### Always

- Never delete files unless explicitly asked
